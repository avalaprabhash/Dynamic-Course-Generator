## Overview

### What is this?

This application helps students, self-learners, and educators create personalized learning experiences. Simply enter a topic (like "Machine Learning" or "Spanish Grammar"), specify how long you want to study, and the AI generates a complete course with:

- Structured modules and lessons
- Learning objectives aligned with Bloom's Taxonomy
- Quizzes that adapt to your skill level
- Progress tracking

### What is Bloom's Taxonomy?

Bloom's Taxonomy is a framework for categorizing educational goals. It describes six levels of cognitive complexity:

```
┌─────────────────────────────────────────────────────────────┐
│                     BLOOM'S TAXONOMY                        │
│                   (Higher-Order Thinking)                   │
│                                                             │
│   ┌─────────┐                                               │
│   │ CREATE  │ ← Produce new or original work                │
│   ├─────────┤                                               │
│   │EVALUATE │ ← Justify decisions and actions               │
│   ├─────────┤                                               │
│   │ ANALYZE │ ← Draw connections among ideas                │
│   ├─────────┤                                               │
│   │  APPLY  │ ← Use information in new situations           │
│   ├─────────┤                                               │
│   │UNDERSTAND│ ← Explain ideas and concepts                 │
│   ├─────────┤                                               │
│   │REMEMBER │ ← Recall facts and basic concepts             │
│   └─────────┘                                               │
│                   (Lower-Order Thinking)                    │
└─────────────────────────────────────────────────────────────┘
```

Courses generated by this app progress through these levels, ensuring a solid foundation before advancing to complex skills.

## Features

### 1. AI-Powered Course Generation
- Enter any topic and duration
- Receive a complete course with modules, lessons, and quizzes
- Content is structured using Bloom's Taxonomy
- Difficulty levels: Beginner, Intermediate, Advanced

### 2. Dynamic Adaptive Quizzes
- Questions adjust difficulty based on your performance AND feedback
- **Quiz Feedback System**: After wrong answers, select:
  - "Too difficult" → moves DOWN one Bloom level
  - "Concept unclear" → stays at same level with simpler questions
  - "Need more examples" → regenerates questions with more examples
- Uses exponential smoothing for fair difficulty transitions
- Bloom level progression: Remember → Understand → Apply → Analyze → Evaluate → Create
- Automatic advancement when consistently scoring well

### 3. Per-User Progress Tracking
- Visual dashboard showing completion status
- Resume functionality - continue where you left off
- Mastery indicators (Beginning → Developing → Proficient → Mastery)
- Per-lesson and overall statistics
- Tracks current lesson position across sessions

### 4. Course Confirmation & Regeneration
- Preview generated courses before starting
- Regenerate courses if not satisfied
- Version history maintained
- Locks lessons until course is confirmed

### 5. User Authentication
- JWT-based authentication
- Secure password hashing with Argon2
- Per-user course progress isolation

### 6. Beautiful, Modern UI
- Clean, responsive design with TailwindCSS
- Intuitive navigation
- Mobile-friendly
- Real-time feedback modals

## Quick Start

### Prerequisites

- **Python 3.9+** (for backend)
- **Node.js 18+** (for frontend)
- **npm or yarn** (package manager)

### Installation

1. **Clone or navigate to the project:**
   ```bash
   cd Quiz-Generator
   ```

2. **Set up the Backend:**
   ```bash
   cd backend
   
   # Create virtual environment (recommended)
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   
   # Install dependencies
   pip install -r requirements.txt
   
   # Create environment file
   cp .env.example .env
   # Edit .env if you want to use real OpenAI API (optional)
   ```

3. **Set up the Frontend:**
   ```bash
   cd ../frontend
   
   # Install dependencies
   npm install
   ```

### Running the Application

**Terminal 1 - Start Backend:**
```bash
cd backend
source venv/bin/activate  # If using virtual environment
uvicorn main:app --reload --port 8000
```

**Terminal 2 - Start Frontend:**
```bash
cd frontend
npm run dev
```

**Open your browser:**
```
http://localhost:5173
```

### Using Real AI (Optional)

By default, the app uses mock data. To use OpenAI:

1. Edit `backend/.env`:
   ```
   USE_MOCK_LLM=false
   OPENAI_API_KEY=your-actual-api-key
   ```

2. Restart the backend server

## Algorithms

### Dynamic Bloom's Taxonomy Adjustment

The system dynamically adjusts the Bloom level based on quiz performance and user feedback:

**Bloom Level Flow:**
```
Remember → Understand → Apply → Analyze → Evaluate → Create
```

**Adjustment Logic:**
- **"Too difficult" feedback**: Move DOWN one Bloom level
  - Example: Apply → Understand
- **"Concept unclear" feedback**: Stay at SAME level, regenerate questions with simpler language
- **"Need more examples" feedback**: Stay at SAME level, regenerate with more examples
- **100% score (3+ questions)**: Move UP one Bloom level
- **80%+ score**: Move UP one Bloom level
- **<50% score**: Move DOWN one Bloom level
- **Otherwise**: Stay at current level

**Example Progression:**
```
User starts at: Remember level
Quiz 1: 100% correct → Move to Understand
Quiz 2: 66% correct, feedback "Too difficult" → Move to Remember
Quiz 3: 90% correct → Move to Understand
Quiz 4: 100% correct → Move to Apply
```

### Adaptive Difficulty (Exponential Smoothing)

We use exponential smoothing to prevent difficulty from swinging wildly after a single good or bad quiz attempt.

```
Formula:  new_score = α × current_result + (1 - α) × previous_score

Where:
  α (alpha) = 0.3
  current_result = quiz score (0-1 scale)
  previous_score = smoothed historical score (0-1 scale)
```

**Why this works:**
- With α = 0.3, historical performance has 70% weight
- This means one bad quiz won't immediately drop you to "easy"
- Consistent performance over time determines your level

**Example:**
```
Previous smoothed score: 0.50 (medium performance)
New quiz score: 0.80 (80% correct)

new_score = 0.3 × 0.80 + 0.7 × 0.50
          = 0.24 + 0.35
          = 0.59

Result: Slight increase, not a jump to "hard" yet
```

### Quiz Feedback System

After each wrong answer, users can provide feedback:

```
┌─────────────────────────────────────┐
│      Why did you get this wrong?    │
├─────────────────────────────────────┤
│  [ ] Too difficult                  │
│  [ ] Concept unclear                │
│  [ ] Need more examples             │
└─────────────────────────────────────┘
```

**Feedback Impact:**
- Stored per question attempt
- Used to determine next Bloom level
- Influences question regeneration strategy
- Does NOT block quiz flow (optional, skippable)

### Difficulty Thresholds

```
If smoothed_score > 0.75:  → Increase difficulty (if not already hard)
If smoothed_score < 0.45:  → Decrease difficulty (if not already easy)
Otherwise:                 → Stay at current difficulty
```

### Mastery Levels

| Score Range | Level | Description |
|-------------|-------|-------------|
| 85%+ | Mastery | Excellent! You've mastered this content |
| 70-84% | Proficient | Good understanding, keep practicing |
| 50-69% | Developing | Making progress, review the material |
| <50% | Beginning | Needs more practice and review |

## Limitations

1. **Rule-Based Adaptation**
   - The "adaptive" system uses rule-based logic, not actual reinforcement learning
   - Feedback improves content through prompt engineering, not model fine-tuning

2. **Perplexity API Required**
   - Live LLM mode requires Perplexity API key
   - Mock mode available for testing without API access

3. **No Offline Support**
   - Requires backend server to be running
   - No PWA/service worker implementation

5. **Limited Quiz Types**
   - Only multiple-choice questions
   - No fill-in-blank, matching, or open-ended questions
